# Portfolio Project Rules

These rules apply to every milestone and every code change.

------------------------------------------------------------------------

## Source of Truth

Before making any changes, always read:

1.  `Quadri_Busari_ATS_Optimized_CV.docx`
2.  `PERSONAL_BRAND.md`

The CV is the factual source of truth.

The PERSONAL_BRAND file defines creative direction.

If information is missing, ask instead of inventing it.

------------------------------------------------------------------------

## Never Invent Information

Do not fabricate:

-   Employers
-   Clients
-   Certifications
-   Awards
-   Dates
-   Metrics
-   Projects
-   Technologies
-   Testimonials

Only use verified information from the project files.

------------------------------------------------------------------------

## Development Workflow

Build incrementally.

Complete one milestone at a time.

Stop after each milestone.

Wait for approval before continuing.

Do not jump ahead.

------------------------------------------------------------------------

## Code Standards

-   Semantic HTML5
-   Modern CSS
-   Vanilla JavaScript (ES6+)
-   Modular architecture
-   Clean folder structure
-   Reusable components
-   Consistent naming
-   Accessibility first
-   Mobile-first responsive design

------------------------------------------------------------------------

## Allowed Libraries

-   GSAP
-   ScrollTrigger
-   Lenis
-   Three.js

Do not introduce React, Vue, Angular, Bootstrap, Tailwind, jQuery, or
unnecessary dependencies unless explicitly requested.

------------------------------------------------------------------------

## Design Principles

The portfolio should feel:

-   Premium
-   Minimal
-   Technical
-   Elegant
-   Fast
-   Trustworthy

Design inspiration:

-   Apple
-   Linear
-   Vercel
-   Stripe
-   Framer

Avoid generic templates.

------------------------------------------------------------------------

Every section must reinforce the professional identity:

**AI Evaluation Specialist**

Highlight:

-   AI Evaluation

-   LLM Evaluation
-   Prompt Engineering
-   AI Safety

------------------------------------------------------------------------

## Quality Checklist

Before finishing any milestone, verify:

-   Accurate content
-   Responsive layout
-   Accessible interactions
-   Clean code
-   Consistent design language
-   Smooth animations
-   Good performance

Refine weak areas before stopping.

------------------------------------------------------------------------

## Working Style

Think like a senior frontend engineer.

Optimize for maintainability and craftsmanship over speed.

When multiple solutions are possible, choose the one that best balances
user experience, readability, accessibility, and long-term
maintainability.
